    计算机网络模型

    Yrenjie 2022/4/1 计算机基础

    # 计算机网络

    # OSI七层网络模型

    OSI七层模型由国际标准化组织提出，其中分为应用层、表现层、会话层、传输层、网络层、数据链路层、物理层。

    1. **应用层：**为用户提供服务的层面
    2. **表现层：**数据处理，例如编解码、加解密、压缩解压缩
    3. **会话层：**为不同程序之间建立会话通信
    4. **传输层：**为不同主机进程之间提供数据传输服务
    5. **网络层：**路由于寻址
    6. **数据链路层：**管理相邻节点的数据通信
    7. **物理层：**网线

    OSI的七层体系结构完善，理论完整，但是太过复杂且不实用，在多层功能中出现重复出现的情况，运行效率不高。

    # TCP/IP四层模型

    TCP/IP的四层网络模型可以看作是OSI七层模型的精简版，由以下4层组成：

    1. 应用层：（应用层、表示层、会话层）
    2. 传输层：（传输层）
    3. 网络层：（网络层）
    4. 网络接口层：（数据链路层、物理层）

    # 应用层

    该层负责处理端与端之间的程序的信息交换，它定义了信息交换格式，我们把应用层交互的数据单元称为报文，信息会交由下一层传输层来传输，这一层的协议有http、ftp、smtp、dns等；

    # 传输层

    该层负责为端与端之间的通信提供数据传输服务，应用层利用该服务传输报文，这一层的协议有TCP、UDP；

    # 网络层

    该层有两个重要任务，一个是将传输层提交过来的报文段或用户数据报封装成分组和包进行传送，另一个是选择合适的路由，使源主机运输层传送下来的分组，能通过网络层的路由器找到目的主机；

    # 网络接口层

    该层分为数据链路层和物理层。

    1. 数据链路层的作用是将网络层传送下来的数据报组装成帧，在两个相邻节点间的链路传输帧
    2. 物理层的作用是实现相邻计算机节点之间的比特流透明传送，尽可能屏蔽掉传输介质和物理设备的差异

    # HTTP vs HTTPS

    两者传输层都是基于TCP为底层协议来实现的，只不过HTTPS相对HTTP来说，增加了SSL/TLS协议来做加密和安全认证的，所以在速度上HTTP更佳，但在安全性上HTTPS更佳；从端口来看，HTTP默认端口为80，HTTPS为443

    # HTTPS核心—SSL/TLS协议

    # SSL与TLS区别

    SSL与TLS没多大区别，在SSL 3.0进一步升级，新版本命名为TLS，但习惯叫法还是SSL/TLS。

    # SSL/TLS协议工作原理

    加密算法主要分为非对称加密和对称加密，SSL/TLS协议核心要素是非对称加密。

    • 非对称加密，主要采用一个公钥，一个私钥。在通信时，私钥仅由解密者保存，公钥由任何一个想与解密者通信的发送者（加密者）所知。

    公钥只能加锁，这样子信息传输过程中保证了安全性，然后由服务端利用私钥进行解密拿到传输信息。

    • 对称加密，由于非对称加密需要大量的计算，导致效率不高，所有在实际中SSL/TLS对消息还是采用了对称加密

    通信双方共享一个密钥，保密性依赖于密钥的保密性。
