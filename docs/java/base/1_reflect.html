    反射及代理

    Yrenjie 2022/4/29 Java反射

    # 反射

    # Class对象

    每当我们编写并编译了一个新类，就会产生一个 Class 对象，它包含了与类有关的信息，Class 对象都属于 Class 类型，既然它也是对象，那我们就可以获取和操控它的引用。

    # 获取方式

    // 通过实例化对象获取class对象
Test test = new Test();
Class clazz = test.getClass();

// 类字面常量
Class clazz = Test.class;

// 通过Class的静态方法forName
Class clazz = Class.forName("Test");

    # 常用方法

    方法 用途
    asSubClass(Class clazz) 把传递的类对象转换成代表其子类的对象
    Cast 把对象转换成代表类或是接口的对象
    getClassLoader() 获取类加载器
    getClasses() 返回一个数组，数组包含类中的所有公共类和接口
    getDeclaredClassed() 返回一个数组，数组包含类中的所有类和接口
    forName(String className) 根据类名获取class对象
    getName() 获得类的完整路径名字
    newInstance() 创建实例
    getPackage() 获取类的包路径
    getSimpleName() 获取类的名称
    getSuperclass() 获取当前类继承的父类名称
    getInterfaces() 获取当前类实现的类或接口

    # 反射

    类 Class 支持反射的概念，java.lang.reflect 库中支持类 Field、Method、Constructor（每一个都实现了 Member 接口），这些类型的对象由 JVM 运行时创建，以表示未知类中的对应成员。通常我们不会直接使用反射，但反射可以用来支持其他 Java 特性，例如对象序列化等

    Field 代表类的成员变量（成员变量也称为类的属性），Class 类中定义了如下方法用来获取 Field 对象

    方法 用途
    getField(String name) 获取某个公有属性对象
    getFields() 获取所有公有属性对象
    getDeclaredField(String name) 获取某个属性对象（包含私有）
    getDeclaredFields() 获取所有属性对象

    Field 类定义了如下方法设置成员变量的信息

    方法 用途
    equals(Object obj) 属性与obj相等返回true
    get(Object obj) 获取属性值
    set(Object obj, Object value) 设置属性值

    如果设置私有属性的值：

    Test test = new Test();
Class clazz = test.getClass();
Field field = clazz.getDeclaredField("test");
// 设置私有属性需开启
field.setAccessible(true);
field.set(test, 1);

    Method 代表类的方法，Class 类中定义了如下方法用来获取 Method 对象

    方法 用途
    getMethod(String name, Class...<?> parameterTypes) 获得该类某个公有的方法
    getMethods() 获得该类所有公有的方法
    getDeclaredMethod(String name, Class...<?> parameterTypes) 获得该类某个方法
    getDeclaredMethods() 获得该类所有方法

    Method 类定义了如下方法对方法进行调用

    方法 用途
    invoke(Object obj, Object... args) 传递 object 对象及参数调用该对象对应的方法

    Constructor 代表类的构造器，Class 类中定义了如下方法用来获取 Constructor 对象

    方法 用途
    getConstructor(Class...<?> parameterTypes) 获得该类中与参数类型匹配的公有构造方法
    getConstructors() 获得该类的所有公有构造方法
    getDeclaredConstructor(Class...<?> parameterTypes) 获得该类中与参数类型匹配的构造方法
    getDeclaredConstructors() 获得该类所有构造方法

    Constructor 代表类的构造方法

    方法 用途
    newInstance(Object... initargs) 根据传递的参数创建类的对象

    # 动态代理

    # 优点

    • 灵活、解耦
    • 方法增强
    • 提高系统扩展性，同时也提高了工作效率

    # 实现方式

    • JDK动态代理

    JDK动态代理需要实现一个处理方法调用的handler，用于实现代理方法的内部逻辑，实现InvocationHandler接口

    handler：

    public class JdkProxyHandler implements InvocationHnadler {
    private Object target;
    public JdkProxyHandler(Object target) {
        this.target = target;
    }
    /**
     * 参数一：代理对象本身。一般不管
     * 参数二：正在被代理的方法
     * 参数三：被代理方法，应该传入的参数
     */
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("处理前");
        // 可以有一些逻辑处理
        Object result = method.invoke(target,args);
        // 可以有一些逻辑处理
        System.out.println("处理后");
        return result;
    }
}

    • Cglib动态代理

    CGLib动态代理技术不需要目标对象实现自一个接口，只需要实现一个处理代理逻辑的切入类，并实现MethodInterceptor接口。

    引入依赖：

    <dependency>
    <groupId>cglib</groupId>
    <artifactId>cglib</artifactId>
    <version>3.3.0</version>
</dependency>

    代理类逻辑处理类：

    public class CglibProxyInterceptor implements MethodInterceptor {
    private Enhancer enhancer = new Enhancer();
    /**
     * 获取代理类
     */
    public Object getProxy(Class clazz) {
        enhancer.setSuperClass(clazz);
        enhancer.setCallback(this);
        return enhancer.create();
    }
    
    public Object intercept(Object object, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
        System.out.println("处理前");
        // 可以有一些逻辑处理
        Object result = methodProxy.invokeSuper(object,args);
        // 可以有一些逻辑处理
        System.out.println("处理后");
        return result;
    }
}

    Cglib的特点：

    • 使用CGLib实现动态代理，完全不受代理类必须实现接口的限制
    • CGLib底层采用ASM字节码生成框架，使用字节码技术生成代理类，比使用Java反射效率要高
    • CGLib不能对声明为final的方法进行代理，因为CGLib原理是动态生成被代理类的子类
